Building the Mini-Application

We use Ant, that comes bundled with J2EE 1.4 Beta Reference Implementation, to build  the source code. Ant is a Java-based build tool like make,   which has been developed  under the auspices of the Apache Software  Foundation.

This section explains how to build the application binaries for the customer mini-application. Building all the application binaries  and  performing  other tasks with Ant involves invoking ant  from <adventure.home>/src/miniapps/customer/src directory of the mini-application installation.

Before building the application, you need to configure the <adventure.home>/src/build.properties file to match your enviroment. You need to set the following properties:
Property Name
 Description
 Examples
j2ee.server.name
 Host name of the server where the J2EE SDK is installed
 localhost
j2ee.server.port
 HTTP port where the J2EE SDK can be accessed
 8000     
j2ee.home
 The absolute path to the directory where J2EE SDK is installed
 c:\j2sdkee1.4 (on Windows) or /home/mydir/j2sdkee1.4   (on  Unix)
Note: On Windows, while writing the paths in the build.properties files,  you need to escape the "\" properly by using "\\" instead. So, for example,  to set j2ee.home to c:\j2sdkee1.4, the entry in the properties file will be j2ee.home=c:\\j2sdkee1.4.

To build the application binaries on Unix, open a shell prompt and enter these commands: 

cd <adventure.home>/src/miniapps/customer/src
ant [argument]
To build the application binaries on Windows, open a shell prompt and enter these commands: 
cd <adventure.home>\src\miniapps\customer\src
ant [argument]
For the above command to work, your PATH environment variable must have been set as explained in the installation instructions and ANT_HOME environment  variable, if any, should not point to any other version of ant your system may have. When ant is invoked without any arguments, all the application  binaries are built. The binaries are placed in the following folders:       
<adventure.home>/src/miniapps/customer/build/customer.ear
