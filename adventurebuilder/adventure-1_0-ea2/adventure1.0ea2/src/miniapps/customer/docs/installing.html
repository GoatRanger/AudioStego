Installing the Mini-Application

This section describes how to install the Customer mini-application

System Requirements and Required Environment Variables

The system requirements and required environment variables are the same as that of the Java  Adventure Builder Demo . Please refer to the system requirements and required environment variables mentioned here

Installing the Mini-Application

Since the Customer mini-application presents a very small subset of the functionalities of Java  Adventure Builder Demo, you must setup the database resources required by the Java  Adventure Builder Demo . Follow steps 1 through 4 under "Installing the Demo" in the Adventure Builder Demo installation instructions listed here

    Run the setup script to deploy the application

    If you are using Unix, open a shell prompt and enter these commands: 

    cd <adventure.home>/src/miniapps/customer/src

    $J2EE_HOME/bin/deploytool -deployModule ../build/customer.ear
    If you are using Windows, open a command prompt and enter these commands: 
    cd <adventure.home>\src\miniapps\customer\src

    %J2EE_HOME%\bin\deploytool -deployModule ..\build\customer.ear
    The application binaries are pre-built and are present in 
    <adventure.home>/src/miniapps/customer/build/customer.ear
To start using the mini-application , please go to http://localhost:8000/customer
